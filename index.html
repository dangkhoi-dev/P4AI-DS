Key Insights: Survival Patterns

  • Gender effect: Females survived at 3.9× the rate of males — the single most discriminative feature in the dataset. Any classifier should learn this boundary first.
  • Class gradient: Survival rate drops from 63% (1st) → 47% (2nd) → 24% (3rd) — reflecting both lifeboat access proximity and crew prioritization.

8. Outlier Detection

Key Insights: Outlier Impact

🔍 Tabular EDA — Summary of Findings

  • Demographic Disparity: Gender and Pclass are the two most powerful predictors. The "women and children first" protocol is clearly reflected in the survival rates, making Sex the #1 feature.
  • Socio-Economic Signal: Fare and Pclass are strongly anti-correlated, both encoding wealth. Together they predict survival more powerfully than either alone — expect interaction terms to boost model performance.
  • Missing Data Strategy: Cabin (77% missing) should be binarized; Age (20% missing) should be imputed using grouped medians by Sex and Pclass; Embarked (2 missing) can use mode-fill.
  • Feature Engineering Opportunities: Title extraction from Name, FamilySize = SibSp + Parch + 1, IsAlone flag, Cabin deck letter, and ticket prefix can all meaningfully improve predictive models.
  • Distribution Shapes: Fare's extreme right-skew demands log-transformation before any distance or gradient-based algorithm is applied.

From EDA Insights to a Trained Classifier

A complete supervised-ML pipeline: preprocess the Titanic data based on EDA findings (Section 1.1), train Random Forest and SVM on the same features, then compare performance side-by-side.

1. Preprocessing Pipeline (driven by EDA)

Every preprocessing step directly addresses a finding from the EDA above — nothing is done arbitrarily.

1
Missing-value imputation

Age ← median(Pclass, Sex); Embarked ← mode ("S"). EDA showed Age distributions differ by Pclass/Sex (Fig TAB-7).

2
Feature engineering

FamilySize = SibSp + Parch + 1; Title extracted via regex from Name; rare titles merged to Rare.

3
Categorical encoding

LabelEncoder on Sex, Embarked, Title. Tree-based models are split-invariant → no one-hot needed.

4
Drop & split

Drop PassengerId, Name, Ticket, Cabin, SibSp, Parch. Stratified 80/20 split with random_state=42.

# Preprocessing — handles NaNs, engineers Title + FamilySize, encodes categoricals
from sklearn.preprocessing import LabelEncoder
df_model = df.copy()

# (1) Impute Age by group-median, Embarked by mode
df_model['Age'] = df_model.groupby(['Pclass','Sex'])['Age']\
    .transform(lambda x: x.fillna(x.median()))
df_model['Embarked'] = df_model['Embarked'].fillna(df_model['Embarked'].mode()[0])

# (2) Feature engineering
df_model['FamilySize'] = df_model['SibSp'] + df_model['Parch'] + 1
df_model['Title'] = df_model['Name'].str.extract(r' ([A-Za-z]+)\.', expand=False)
rare = ['Lady','Countess','Col','Capt','Dr','Rev','Sir','Major','Don','Jonkheer','Dona']
df_model['Title'] = df_model['Title'].replace(rare, 'Rare')\
    .replace({'Mlle':'Miss','Ms':'Miss','Mme':'Mrs'})

# (3) Encode categoricals
le = LabelEncoder()
for c in ['Sex','Embarked','Title']:
    df_model[c] = le.fit_transform(df_model[c])

# (4) Drop leakage / unused columns
df_model = df_model.drop(['PassengerId','Name','Ticket','Cabin','SibSp','Parch'], axis=1)

Takeaways — Tabular ML

5. Class Distribution & Imbalance
Fig 5.1 – Class Frequency & Proportion. 4,825 Ham (86.6%) vs 747 Spam (13.4%). A ~6.5:1 imbalance ratio that will cause naive models to predict "Ham" for everything and still achieve 86% accuracy — requiring careful evaluation with precision/recall/F1.

Key Insights: Class Imbalance

  • The severe imbalance (~87% Ham) means accuracy is a misleading metric — a model predicting "Ham" for every message achieves 86.6% accuracy but catches 0% of spam.
  • Recommended strategies: SMOTE oversampling of the Spam class, class-weight adjustment in sklearn (class_weight='balanced'), or threshold tuning on predicted probabilities.
  • Evaluation should prioritize Recall for Spam class — false negatives (missed spam) are more costly than false positives (ham flagged as spam).

  • Average word length: Spam có xu hướng dùng từ ngắn hơn (txt, ur, free, win) — thông điệp được tối ưu hóa để đọc nhanh trên màn hình nhỏ.
  • TTR thấp của Spam là bằng chứng định lượng cho thấy spam messages được sinh ra từ template cố định với một số slot thay thế — điều này giải thích tại sao rule-based filters hoạt động tốt.

💬 Text EDA — Summary of Findings

  • Class Imbalance Critical: 86.6% Ham vs 13.4% Spam demands imbalance-aware strategies. Use SMOTE or class_weight='balanced'; evaluate with F1-score for Spam, not overall accuracy.
  • Length is a Free Feature: Message word count and character count alone achieve solid classification baselines. These should always be included as explicit features alongside TF-IDF.
  • TF-IDF is Highly Effective: The near-orthogonal vocabulary distributions (cosine sim ≈ 0.0075) mean even simple linear models with TF-IDF features will achieve excellent results (>97% F1).
  • Bigrams Add Context: Phrase-level patterns ("free entry", "reply stop") capture advertising language patterns that single tokens cannot. Always include at least bigrams in the n-gram range.
  • Preprocessing Pipeline: Lowercase → remove punctuation → tokenize → remove stopwords → TF-IDF (1,2)-gram → dimensionality reduction (optional SVD). This standard pipeline is well-suited to this dataset.

import tensorflow as tf
from tensorflow.keras.applications.resnet50 import preprocess_input

(X_train, y_train), (X_test, y_test) = tf.keras.datasets.cifar10.load_data()
IMG_HEIGHT, IMG_WIDTH = 224, 224

def preprocess_image(image, label):
    image = tf.image.resize(image, (IMG_HEIGHT, IMG_WIDTH))
    image = preprocess_input(image)           # BGR + ImageNet means
    return image, label

train_ds = (tf.data.Dataset.from_tensor_slices((X_train, y_train))
            .map(preprocess_image).batch(32).prefetch(tf.data.AUTOTUNE))

Fig 10.1 – Jaccard Similarity giữa các cặp captions cho cùng một ảnh (500 ảnh). Histogram bên trái: tất cả các cặp pairwise. Bên phải: trung bình mỗi ảnh. Mean Jaccard ≈ 0.15–0.20 cho thấy annotators chia sẻ ~15–20% vocabulary.

Key Insights: Annotation Diversity

  • Jaccard similarity thấp (~0.15–0.20) là bình thường và mong đợi — mỗi annotator chọn tập trung vào chi tiết khác nhau (màu sắc, hành động, đối tượng, bối cảnh).
  • Đây là tính năng quý giá của Flickr8k: 5 captions đa dạng cho cùng 1 ảnh cung cấp nhiều training signals hơn so với 1 caption duy nhất, giúp model học vocabulary phong phú hơn.
  • Ảnh có per-image agreement thấp (Jaccard < 0.1) thường là ảnh phức tạp với nhiều đối tượng/hành động — mỗi annotator chú ý đến một phần khác nhau của ảnh.

Key Insights: Ngôn ngữ mô tả hình ảnh

  • Noun-dominant: Captions về ảnh chủ yếu là danh từ (man, woman, dog, field, street...) — mô tả đối tượng là trọng tâm chính.
  • Verb và Adjective tiếp theo: verbs mô tả hành động (running, sitting, playing), adjectives mô tả màu sắc và đặc điểm (white, black, large, young).

  • POS distribution này là đặc trưng của image description language, khác hẳn với conversational text (SMS Spam) hay formal text — cần pretrained language model phù hợp (CLIP text encoder).

🔗 Multimodal EDA — Summary of Findings

  • Cross-Modal Quality: Both image and text modalities exhibit high internal consistency — variable-resolution clean images paired with well-formed English captions that follow consistent length guidelines. Annotation quality appears high.
  • Image Preprocessing: Aspect-ratio-preserving resize to 224×224 + per-channel normalization using ImageNet statistics. Filter out images smaller than 50px to avoid low-information samples destabilizing batch normalization.
  • Caption Preprocessing: Lowercase → punctuation removal → tokenization → max_length=25 truncation → padding. Remove or flag captions with anomalous char/word ratios.
  • Human-Centric Dataset: The vocabulary is dominated by people-related nouns and action verbs — indicating this is likely derived from a general-purpose photography dataset (Flickr-30K style or MS-COCO). Models benefit strongly from person detection priors.
  • Modeling Direction: Pre-trained vision-language models (CLIP, BLIP, LLaVA) will dramatically outperform training from scratch given this dataset's alignment with ImageNet/Common Crawl pre-training distributions. Fine-tuning a pre-trained model is the recommended approach.
